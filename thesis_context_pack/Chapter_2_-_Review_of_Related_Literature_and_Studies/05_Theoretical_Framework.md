# Theoretical Framework

This investigation taps into computer vision, low-light imaging, environmental monitoring via sensors, IoT communication, and real-time system design theories. A theory of computer vision helps to illustrate a facial recognition system through steps like image acquisition, feature extraction, and pattern classification that ultimately allow separating authorized users from intruders. The theory of low-light imaging backs up night-vision operation via infrared illumination, noise abatement, and contrast enhancement. Fire detection is a cell of sensor signal processing theory that decodes changes in smoke concentration and visual cues to identify the presence of a fire-risk situation. Multi-sensor inputs have a synergistic effect on the accuracy and reliability of detection, explained by the theories above.
In particular, IoT communication theory explains how lightweight network protocols support timely event transfer and alert delivery in distributed monitoring systems. Prior works commonly discuss publish-subscribe patterns such as MQTT; however, the current implementation uses HTTP POST + JSON for ESP32-C3 sensor-to-server messaging. In the implemented architecture, Raspberry Pi 2 handles local server-side services while a second-hand laptop handles face-recognition and flame-detection processing. Previous research has highlighted constraints such as reliance on a single sensor, low-light performance degradation, and separation of security and fire detection functionalities. These shortcomings guided the present framework, which fuses facial recognition, night-vision imaging, sensor-based fire detection, and edge communication in one platform.
The framework sees the system as a practical embodiment of interconnected sensing, automated decision-making, and real-time digital communication.
