# Matrix of Methods

| Stage | Research Setting | Respondents of the Study | Data Gathering Procedures | Data Gathering Instruments | Statistical Analysis |
| --- | --- | --- | --- | --- | --- |
| **Phase 1. System Context, Scope, and Architecture Design** | University laboratory (design/planning) + condo-like layout review | No human respondents; system context and component set | Review related works, define 2-area scope (Living Room and Door Entrance Area), produce architecture and deployment plan for Raspberry Pi 2 server host + second-hand laptop vision host setup | Layout sketches, architecture diagrams, component specifications | Qualitative design output only |
| **Phase 2. Development of Core Detection Modules** | University laboratory/workshop with controlled tests | Core modules (face, flame, smoke events, door-force events) | Implement and calibrate OpenCV face recognition, indoor flame detection, HTTP sensor ingestion (`POST /api/sensors/event`), and door-force event handling | Annotated image/video sets, sensor test scenarios, module logs, timing scripts | Descriptive module metrics (accuracy/detection rates/latency summaries) |
| **Phase 3. System Integration and Multi-Sensor Intelligence Layer** | Lab integration then condo-like integrated deployment | Integrated prototype (Raspberry Pi 2 server host; second-hand laptop vision host; indoor USB camera; outdoor ESP32-CAM; `mq2_living`, `mq2_door`, `door_force`) | Integrate camera inputs + sensor events, apply fusion logic, validate alerting, logging, and dashboard behavior under controlled scenarios | Integration logs, event logs, snapshots, dashboard evidence | Descriptive verification of integration correctness |
| **Phase 4. Iterative Testing, Adjustment, and Refinement** | Condo-like/simulated unit | Integrated system under repeated scenario tests; optional small user feedback group | Run repeated scenarios (normal, intruder, fire-related, nuisance), record false alarms/misses/latency, tune thresholds and rules, rerun tests | Scenario scripts/checklists, logs, recordings, parameter records, optional short feedback forms | Descriptive comparison across iterations |
| **Phase 5. Final Performance Evaluation and Analysis** | Condo-like environment with final calibrated build | Final integrated system + optional human evaluators | Conduct repeated final trials, compute detection and latency outcomes, summarize optional user ratings on alert clarity/timeliness/reliability | Timestamped system logs, classification records, latency summaries, questionnaires | Final descriptive statistics (and simple inferential tests only if user-rating data is analyzed) |
