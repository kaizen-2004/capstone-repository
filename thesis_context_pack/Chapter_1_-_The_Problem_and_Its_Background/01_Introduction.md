# Introduction

Contemporary security and safety systems increasingly combine Internet of Things (IoT) devices, computer vision, and local monitoring software to produce faster and more informative alerts. Recent literature shows that face recognition, fire-related sensing, and networked alerting can be integrated into practical prototype systems using low-cost edge hardware, night-vision imaging, and lightweight communication protocols. These developments suggest that residential monitoring can move beyond passive recording toward active event classification and timely warning.

The review of literature also shows, however, that many existing systems still face important limitations. Some rely too heavily on a single cue, such as motion, image color, or isolated smoke readings. Others work well only in controlled lighting, or they produce unnecessary alerts when environmental conditions change. In many cases, intrusion monitoring and fire monitoring are implemented separately, which leads to fragmented logs, slower operator response, and less reliable decision-making. These limitations justify the development of a unified prototype that can combine visual evidence, environmental sensing, and local alert management in a single workflow.

This study focuses on the development of a real-time intruder and fire monitoring alert system for a condo-like environment. The implemented scope covers two monitored areas: the Living Room and the Door Entrance Area. The system uses camera inputs with night-vision support, smoke-sensing nodes, and a door-force sensing node to observe security- and fire-related conditions. Sensor events are transmitted through HTTP POST with JSON payloads to `POST /api/sensors/event`, while camera inputs support face-recognition and fire-related visual analysis. The prototype also includes local logs, summaries, alert acknowledgment, and optional remote notification features.

Within the architecture discussed in this paper, Raspberry Pi 2 is retained as an optional deployment component for local edge hosting. This allows the study to remain aligned with embedded-system deployment goals while preserving flexibility in how local monitoring, storage, and visual processing services are finally arranged during testing and presentation. The emphasis of the research is therefore placed on system capability, integration logic, and measured performance rather than on one rigid host-device arrangement.

The need for proactive monitoring remains significant. Fire incidents and property-related threats continue to create risk for households, especially in settings where immediate local awareness and verification are important. A system that can distinguish authorized from non-authorized presence, detect smoke-triggered fire conditions, and present actionable alerts through a local monitoring interface can contribute to faster decision-making and reduced false alarms. For this reason, the present study proposes an integrated, event-driven, and night-vision-capable monitoring prototype as a practical response to current residential safety needs.
