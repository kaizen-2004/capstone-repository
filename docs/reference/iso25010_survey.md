# ISO/IEC 25010 User Evaluation Survey

This questionnaire evaluates the prototype system using all 8 ISO/IEC 25010 product quality characteristics.

## Respondent Groups

Use the respondent groups defined in your study significance/final participant list (for example: household-member proxies, students, faculty evaluators).

## Instructions

Please evaluate each statement based on your actual observation and use of the prototype during the demonstration/testing.

## Rating Scale (5-Point Likert)

- `5` Strongly Agree
- `4` Agree
- `3` Neutral
- `2` Disagree
- `1` Strongly Disagree

## Survey Items

### A. Functional Suitability

1. The system provides the required functions for intruder and fire monitoring.
2. The generated alerts are relevant to actual detected events.
3. The dashboard provides sufficient information for monitoring and response.

### B. Performance Efficiency

4. The system responds quickly when an event occurs.
5. Camera view, logs, and alerts load at an acceptable speed.
6. The system remains responsive during continuous operation.

### C. Compatibility

7. The hardware and software components work together properly.
8. Data flow between detection, logging, and alerting modules is consistent.
9. The system works properly on available devices/browsers used in testing.

### D. Usability

10. The interface is easy to learn.
11. Alerts and status information are easy to understand.
12. Acknowledging and reviewing alerts is easy to perform.

### E. Reliability

13. The system operates stably during repeated tests.
14. The system recovers properly after temporary interruptions.
15. Outputs are consistent when the same test scenario is repeated.

### F. Security

16. Access to important system functions appears restricted to authorized users.
17. Logged events and records appear protected from accidental tampering.
18. The prototype provides acceptable protection for data and access in this study context.

### G. Maintainability

19. Thresholds and settings can be adjusted without major rework.
20. Logs and status indicators help identify faults quickly.
21. Modifying one component does not severely break other components.

### H. Portability

22. The system can be moved to another compatible host with manageable effort.
23. Deployment from lab setup to condo-like setup requires only minor adjustments.
24. Replacing equivalent sensors/cameras can be done with limited software changes.

## Optional Open-Ended Questions

1. What part of the system is most useful during monitoring?
2. What issue did you observe most often during testing?
3. What one improvement should be prioritized before deployment?
